# AI Development Rules

IMPORTANT:
- Make only minimal targeted changes
- Do not refactor unrelated code
- Do not rename variables/functions unnecessarily
- Preserve backward compatibility
- Do not remove existing logic unless explicitly requested
- Never modify auth flow unless task requires it
- Never modify GraphQL schema unless instructed
- Keep multi-organization support intact
- Keep existing APIs compatible
- Explain risky changes before applying
- Prefer patching over rewriting
- Avoid touching unrelated files
- Ask before large architectural changes
